Title: fix(security): correct csrf middleware order
Branch: bugfix/fix-csrf-ordering

# Summary

This PR addresses a security bug where CSRF protection was rendered non-functional because the CSRF middleware was placed before `cookieParser()` in `server.js`.

## Related Issue

Fixes #206

## Type of Change

- [x] Bug Fix
- [x] Security Enhancement

## Changes Implemented

- Swapped the order of `app.use(cookieParser())` and `app.use(csrfProtection)` in `server.js` so that cookies are parsed before the CSRF middleware attempts to read the secret cookie.

## Technical Details

### Backend
- `server.js`: Adjusted middleware ordering. This allows `csurf` to correctly retrieve and validate CSRF tokens for all state-changing API endpoints.

## Testing

### Manual Testing

- [x] Verified that state-changing API requests successfully validate the CSRF token now that the cookie parser runs first.

## Security Review
- Resolved CSRF token validation failure, securing the application against Cross-Site Request Forgery attacks.
