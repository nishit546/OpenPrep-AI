Title: fix(security): protect uploads folder
Branch: bugfix/protect-uploads-route

# Summary

This PR addresses a critical security vulnerability where uploaded files (like PDFs and images for Notes and PYQs) were accessible to anyone via the static `/uploads` route without authentication.

## Related Issue

Fixes #205

## Type of Change

- [x] Bug Fix
- [x] Security Enhancement

## Changes Implemented

- Removed the `express.static` serving for `/uploads`.
- Replaced it with an authenticated `GET /uploads/:filename` endpoint.
- Added ownership and privacy verification (using the `Note` and `PYQ` database models) before serving the requested file.

## Technical Details

### Backend
- `server.js`: Implemented custom route to stream files securely with `res.sendFile()`. Included the `protect` middleware from `auth.js` to ensure the user is logged in, and queries to ensure the user has the rights to view the file.

## Testing

### Manual Testing

- [x] Tested uploading a note and attempting to access the file via `/uploads/:filename` from a logged-out session (returns 401).
- [x] Tested accessing another user's private file (returns 403).

## Security Review
- Ensured uploaded files are not publicly accessible without authentication.
- Maintained access for public files and files owned by the requesting user.
